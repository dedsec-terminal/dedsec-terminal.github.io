Noma has introduced Noma Agent Access Control, a solution designed to provide visibility and access governance for AI agents and Model Context Protocol (MCP) servers within an enterprise. This launch aims to address the rapid proliferation of AI agents and MCP servers, which has outpaced existing governance frameworks. As a result, organizations are struggling to manage and secure these agents, which often have access to sensitive data and can execute actions on behalf of users.

Noma Agent Access Control offers a two-layered approach to governing AI agents. The first layer, Registry and Access Control, provides a dynamic registry of all agents and MCP servers, along with their connections and security policies. This layer enables security teams to define access policies, assign identities to agents, and control tool-level access. The second layer, Runtime Enforcement, involves monitoring agent behavior in real-time to detect potential security threats, such as prompt injection attacks or data exfiltration. Noma's AI Detection and Response (AI-DR) platform plays a crucial role in this layer, providing continuous monitoring and detection of security risks.

The combination of Noma Agent Access Control and AI-DR provides organizations with a comprehensive governance framework for AI agents and MCP servers. By defining access policies and monitoring agent behavior in real-time, security teams can ensure that agents operate within established boundaries and detect potential security threats. This approach enables organizations to maintain visibility and control over their AI agents, reducing the risk of security breaches and data misuse. With Noma's solution, security teams can govern AI agents more effectively, ensuring that these agents operate securely and in accordance with organizational policies.
