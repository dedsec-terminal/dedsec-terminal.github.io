Legit Security has introduced new remediation agents that utilize agentic AI to prioritize and fix application security (AppSec) issues. These agents can independently identify vulnerabilities, generate fixes, open pull requests, and confirm results using context learned from each organization's unique codebase. This approach is critical in today's AI-driven development landscape, where attackers can exploit vulnerabilities faster than ever, and rapid remediation is essential.

The need for automated and intelligent AppSec tools has become increasingly important due to the rise of AI-first development. AI-generated code contains more vulnerabilities than human-written code, and the median time to remediate a vulnerability is nearly six times longer than attackers need to exploit it. Legit's remediation agents address this challenge by offering AI-speed remediation centered on the context of the business and codebase. Unlike general-use AI coding tools, Legit's agents have the security knowledge and business context to generate production fixes, rather than patches.

The key features of Legit's remediation agents include unified risk posture, prioritization of issues based on factors such as reachability and exploitability, and the ability to close complete attack surface gaps by opening pull requests across every affected repository in parallel. The agents also validate fixes before opening a pull request and create auditable records of their activity. According to Legit's CEO, Roni Fuchs, security teams are not losing the war due to a lack of talent, but because the model has changed completely, and AppSec testing tools have stayed the same. Legit's remediation agents are designed to address this gap and provide trustable, automated, and intelligent AppSec remediation.
