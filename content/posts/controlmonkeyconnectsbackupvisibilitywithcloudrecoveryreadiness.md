ControlMonkey has introduced a new capability called Data Backup Correlation, which extends its Cyber Resilience Platform to connect data backup posture with cloud configuration recovery. This feature aims to address the lack of visibility into data backup coverage and available recovery points across critical data sources, making it challenging for organizations to understand what data assets are recoverable in case of an incident. The initial release supports AWS Backup and Azure Backup, with plans to expand to other cloud-native backup services.

The Data Backup Correlation capability alerts teams to changes in backup coverage, retention policy violations, missing recovery points, and potential RPO target risks. It also helps prioritize recovery efforts based on recovery risk and business impact, identifying missing backups, monitoring RPO compliance, and detecting cross-region or cross-account protection gaps. This feature provides a complete operational picture of what existed before an incident, what is protected, what is missing, and what should be restored first. By connecting data backup posture with cloud configuration recovery, ControlMonkey gives CISOs and cloud teams the context they need to understand what is protected, what is exposed, and what to recover first.

The Data Backup Correlation capability expands ControlMonkey's Cyber Resilience Platform, which helps organizations discover, govern, back up, and recover cloud and SaaS configuration. By adding visibility into data backup posture, ControlMonkey provides a Unified Control Plane for Resilience, giving organizations a broader recovery picture across both configuration and data protection systems. This new feature enables teams to see backup coverage across critical cloud data sources, review available recovery points, identify unprotected data assets, and detect data sources breaching defined RPO targets, ultimately helping organizations protect and recover their cloud and SaaS configuration more effectively.
