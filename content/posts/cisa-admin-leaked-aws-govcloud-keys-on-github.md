A contractor for the US Cybersecurity and Infrastructure Security Agency (CISA) accidentally leaked highly sensitive credentials, including AWS GovCloud keys and internal system passwords, on a public GitHub repository. The repository, named "Private-CISA," was created in November 2025 and contained a vast number of internal CISA credentials, including cloud keys, tokens, plaintext passwords, logs, and other sensitive assets.

The leak was discovered by a security researcher, Guillaume Valadon, who notified CISA and a cybersecurity blog, KrebsOnSecurity. The exposed credentials included administrative access to three Amazon AWS GovCloud servers and plaintext usernames and passwords for dozens of internal CISA systems.

The contractor, who worked for a government contractor called Nightwing, had disabled GitHub's default setting that blocks users from publishing sensitive information, such as SSH keys, in public code repositories. The leak is considered one of the most egregious government data leaks in recent history, and it raises concerns about CISA's cybersecurity practices.

CISA has acknowledged the incident and is investigating, but the agency claims that there is no indication that any sensitive data was compromised. The GitHub repository was taken offline after the leak was reported, but the exposed AWS keys remained valid for another 48 hours.

The incident has sparked criticism of CISA's cybersecurity practices, particularly given the agency's role in protecting the country's critical infrastructure from cyber threats. The leak is also notable because CISA is currently operating with reduced budget and staffing levels, which may have contributed to the incident.

The incident highlights the importance of proper cybersecurity hygiene, including the use of secure passwords, two-factor authentication, and regular security audits. It also raises questions about the accountability of government agencies and contractors when it comes to cybersecurity incidents.
