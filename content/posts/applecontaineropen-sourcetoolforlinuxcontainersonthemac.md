Apple has released an open-source tool called Apple Container, which allows developers to run Linux containers on Macs with Apple silicon. The tool creates and runs Linux containers as lightweight virtual machines, and it works with OCI-compatible images, making it easy to pull from and push to standard registries. Written in Swift and optimized for Apple silicon, Apple Container provides a lightweight and efficient way to run Linux workloads on Macs.

The latest release of Apple Container, version 1.0.0, introduces several new features and improvements. One of the key additions is the "container machine" feature, which allows developers to create long-lived Linux environments with tight integration into the host Mac. The release also includes a new command for moving files between the host and a running container, as well as fixes for accounting errors and networking issues. Additionally, the configuration system has been moved to a TOML file, replacing the older system settings.

The Apple Container project has gained significant interest and attention from the developer community, with over 46,500 stars and nearly 1,500 forks on GitHub. The tool is available for free on GitHub and can be installed using a signed package from the releases page. To run Apple Container, a Mac with Apple silicon and macOS 26 is required, and the project depends on virtualization and networking features introduced in that release. With its growing popularity and active development, Apple Container is a promising tool for developers looking to run Linux containers on their Macs.
