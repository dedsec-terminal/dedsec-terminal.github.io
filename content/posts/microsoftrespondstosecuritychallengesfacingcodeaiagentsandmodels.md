Microsoft has introduced a range of security tools and capabilities to address the challenges facing code, AI agents, and models. The updates include a multi-agent vulnerability discovery system, new controls for managing and securing AI agents, data protection capabilities, and tools designed to identify potentially vulnerable or compromised AI models before deployment. This move aims to enhance the security of AI-driven systems and protect against potential threats.

The company has expanded the preview of MDASH, a multi-model agentic vulnerability discovery system that integrates with Microsoft Defender. MDASH uses over 100 specialized AI agents and multiple AI models to discover vulnerabilities, validate findings, and assess exploitability in software codebases. Additionally, Microsoft has introduced an integration between Microsoft Defender and GitHub Code Security, which adds production context to vulnerabilities discovered in source code. This integration enables developers to remediate issues using AI-assisted fixes.

Microsoft has also introduced new security controls for AI agents, including the Agent 365 SDK, which adds observability, access control, and compliance features. The company has also announced the availability of Purview, which provides data protection for AI agents, including data exfiltration protections and risk detection. Furthermore, Defender AI model scanning is a new preview capability designed to inspect AI models before deployment, identifying potentially vulnerable or compromised models in registries, workspaces, and CI/CD pipelines. These updates demonstrate Microsoft's commitment to addressing the security challenges associated with AI-driven systems.
