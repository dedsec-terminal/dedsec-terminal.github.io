New Relic has introduced AI Coding Observability, an open-source tool designed to monitor AI-assisted software development workflows. This tool extends monitoring into the software development process, enabling organizations to track, analyze, and audit AI-assisted coding activities. By doing so, it addresses the lack of visibility into AI coding assistants, which often operate outside existing observability systems, limiting visibility into their use.

The introduction of AI Coding Observability aims to bring enterprise-grade rigor to the AI coding phase, where AI coding tools are producing more code than ever. With Gartner predicting that 90% of enterprise software engineers will use AI code assistants by 2028, New Relic's tool is designed to future-proof development strategies and introduce a unified, vendor-neutral pane of glass that normalizes telemetry across major AI coding assistants. This will allow organizations to correlate AI coding activities with existing production infrastructure seamlessly.

The key features and benefits of New Relic AI Coding Observability include gaining insights into code development actions, exercising cost control, enhancing productivity metrics, ensuring security and compliance, and providing strong code transparency. The tool will support multiple coding assistants, including Claude Code, Cursor, GitHub Copilot, Windsurf, and Amazon Q, and will be available as an open-source feature on June 23 at no additional cost. By providing a unified and vendor-neutral solution, New Relic AI Coding Observability aims to remove barriers to quality innovation and help organizations scale their AI-assisted software development workflows with confidence.
