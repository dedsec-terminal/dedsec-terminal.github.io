GitLab 19.0 has been released with several new features, including expanded secrets management, AI workflows, and self-hosted model support. The update aims to address the "AI Paradox" where the surrounding workflows for securing credentials, reviewing and merging changes, and enforcing pipeline standards have not kept pace with the increased use of AI in code development. GitLab 19.0 advances the platform's capabilities by embedding these features where teams already work, reducing handoffs between writing code and shipping it.

The new features include GitLab Secrets Manager, which stores credentials inside the platform and provides access control and audit logging. Developer Flow has also been extended across the full merge request lifecycle, allowing teams to address reviewer feedback, resolve conflicts, and implement features at any stage. Additionally, Components Analytics provides visibility into which CI/CD Catalog components are running across the organization, and which versions are in use. These features are available for Free, Premium, and Ultimate tier users, with some capabilities limited to higher tiers.

GitLab 19.0 also includes support for self-hosted open-source models, allowing teams to run AI models on-premises or in private clouds. The platform now supports four additional open-source models, which were evaluated against task requirements such as multi-step tool use and code generation quality. The update also strengthens software supply chain visibility with dependency scanning and security configuration profiles, giving teams more control over governing what ships and who can access the platform. These features aim to help teams move fast on AI without losing control of what ships, and to provide a more secure and trustworthy development environment.
