**GitHub Breach: Employee Device Hack Leads to Exfiltration of 3,800+ Internal Repos**

GitHub, a Microsoft-owned subsidiary, has announced that it is investigating an unauthorized access to its internal repositories after a threat actor known as TeamPCP listed the platform's source code and internal organizations for sale on a cybercrime forum. The breach occurred due to a compromised employee device, which was infected with a poisoned Microsoft Visual Studio Code extension.

**Key Findings:**

1. **Exfiltration of internal repositories**: The attacker exfiltrated approximately 3,800 internal GitHub repositories, which may include sensitive information such as source code, internal tools, and security protocols.
2. **No customer data affected**: GitHub claims that there is no evidence of impact to customer information stored outside of its internal repositories.
3. **Poisoned VS Code extension**: The breach occurred due to a compromised employee device, which was infected with a poisoned Microsoft Visual Studio Code extension.
4. **TeamPCP's involvement**: The threat actor TeamPCP is behind the breach and has listed GitHub's source code for sale on a cybercrime forum for a minimum of $50,000.
5. **LAPSUS$ cybercrime group involvement**: The LAPSUS$ cybercrime group has teamed up with TeamPCP to jointly sell GitHub repositories for $95,000.

**Additional Incidents:**

1. **Durabletask PyPI package compromise**: TeamPCP has compromised the durabletask PyPI package, which is an official Microsoft Python client for the Durable Task workflow execution framework.
2. **Mini Shai-Hulud malware campaign**: TeamPCP's self-replicating malware campaign, known as Mini Shai-Hulud, continues to expand in reach, with the compromise of the durabletask package being the latest incident.

**Recommendations:**

1. **Monitor infrastructure**: GitHub is closely monitoring its infrastructure for follow-on activity and will notify customers via established incident response and notification channels if any impact is discovered.
2. **Rotate critical secrets**: GitHub has rotated critical secrets as a risk mitigation measure.
3. **Prioritize highest-impact credentials**: GitHub is prioritizing highest-impact credentials to minimize potential damage.

Overall, the breach highlights the importance of robust security measures, including employee device security, to prevent unauthorized access to sensitive information.
