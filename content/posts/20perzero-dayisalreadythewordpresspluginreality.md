Researchers from TrendAI and CHT Security have developed a pipeline that uses AI-driven static analysis and automated dynamic verification to discover zero-day vulnerabilities in WordPress plugins. The system was able to find over 300 critical vulnerabilities in just 72 hours, with each finding manually verified by the researchers. The cost of discovering each vulnerability was estimated to be around $20, which is a significant development in the security industry.

The pipeline's findings include a range of vulnerability classes, such as pre-authentication remote code execution, SQL injection, and privilege escalation. One notable example is a pre-auth RCE vulnerability found in a plugin with over 1,000 GitHub stars. The AI agent was also able to chain together multiple vulnerabilities to create a working attack, without any human guidance. The researchers believe that this technology is already being used by both white-hat and black-hat actors, and that the price floor for discovering vulnerabilities has already been crossed.

The discovery of these vulnerabilities has put a strain on the disclosure infrastructure, with many organizations struggling to keep up with the volume of AI-generated reports. The researchers believe that the solution to this problem is to use more automation, such as AI-assisted triage, to help prioritize and verify the most critical vulnerabilities. However, there are still limitations to the technology, such as the need for human judgment in certain cases, and the difficulty of exploiting vulnerabilities that require specific environmental conditions. Despite these limitations, the researchers expect to see a significant increase in the number of disclosed vulnerabilities in the coming months, and a shift towards more automated and invite-only disclosure programs.
